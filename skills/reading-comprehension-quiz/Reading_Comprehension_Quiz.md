# Reading_Comprehension_Quiz (閱讀理解測驗產生器)

## 1. 技能描述 (Description)
本技能用於根據任意閱讀文本（成語故事、記敘文、說明文等），自動產出結構嚴謹、認知層次分明的閱讀理解測驗題目。所有題目必須遵循素養導向命題原則，並符合國小學生的認知發展水準。

## 2. 觸發條件 (Trigger)
- 當需要為某段文本（故事、比喻、定義）產出閱讀理解測驗題目時。
- 當需要驗證學生是否真正理解了一段內容，而非僅是記憶表面文字時。

## 3. 核心框架：布魯姆認知層次 (Bloom's Taxonomy)

每一組測驗題目（通常 3 題為一組）**必須涵蓋至少兩個不同的認知層次**，以確保測驗的深度與鑑別度。

| 層次 | 英文 | 說明 | 題目範例（以「畫蛇添足」為例） |
|---|---|---|---|
| L1 記憶 | Remember | 從文本中直接提取事實 | 「故事中，第一個畫好蛇的人做了什麼多餘的事？」 |
| L2 理解 | Understand | 歸納、解釋、轉述文意 | 「『畫蛇添足』這個成語主要告訴我們什麼道理？」 |
| L3 應用 | Apply | 將概念套用到新情境 | 「下列哪一個行為最像『畫蛇添足』？」 |
| L4 分析 | Analyze | 拆解結構、辨別因果 | 「為什麼其他人會搶走他的酒？關鍵原因是什麼？」 |
| L5 評鑑 | Evaluate | 做出判斷與批判 | 「你覺得畫好蛇的人是因為自信還是驕傲？為什麼？」 |
| L6 創造 | Create | 產出新觀點或解決方案 | 「如果你是故事中的人，你會怎麼做？」 |

### 3.1 國小階段建議分佈
| 題號 | 建議層次 | 理由 |
|---|---|---|
| 第 1 題 | L1 記憶 | 建立信心，確認學生有讀完文本 |
| 第 2 題 | L2 理解 | 測試是否真正理解核心概念 |
| 第 3 題 | L3 應用 | 將概念遷移到生活情境，這是素養的核心 |

> ⚠️ **重要原則**：第 3 題是區分「記住了」和「學會了」的關鍵。
> 如果學生只背了定義但沒理解，第 3 題就會答錯。

## 4. 命題規範 (Design Rules)

### 4.1 題幹 (Stem) 規範
| 規則 | 說明 | ❌ 錯誤範例 | ✅ 正確範例 |
|---|---|---|---|
| 正面表述 | 避免否定句型 | 「下列哪個**不是**...」 | 「下列哪個**最符合**...」 |
| 單一觀念 | 每題只測一個核心概念 | 「...的意思和用法是？」 | 「...的意思是？」 |
| 去暗示化 | 避免題目間互相洩漏答案 | (第 2 題的選項直接出現在第 1 題裡) | 每題獨立，互不參照 |
| 情境化 | L3 應用題必須提供具體生活場景 | 「請應用此成語」 | 「小明考試寫完了還硬加...」 |

### 4.2 選項 (Options) 規範
| 規則 | 說明 |
|---|---|
| 數量固定 | 每題 **4 個選項** (A/B/C/D) |
| ❗ 字數平衡 | **所有選項字數差距不得超過 ±3 字**。這是硬性規則，絕對不可突破。過去的經驗證明，正確答案往往是最長的選項，這會讓學生學到「選最長的」而不是「選對的」。|
| 詞性一致 | 如果答案是名詞，其他選項也必須是名詞 |
| 似真性 | 誘答項必須看起來「像是對的」，不能一眼識破 |
| 互斥性 | 正確答案必須是唯一且絕對正確的 |
| 隨機排序 | 正確答案的位置 (A/B/C/D) 必須隨機分佈 |

> 🚨 **絕對禁止**：正確答案是四個選項中最長的。這是經實戰驗證的重大命題缺陷。
> 如果無法縮短正確答案，就必須加長誘答項，使四個選項字數接近。

### 4.3 誘答項 (Distractor) 設計原則
誘答項的品質決定了整個測驗的鑑別度。

| 策略 | 說明 | 範例 |
|---|---|---|
| 常見迷思 | 根據學生常犯的錯誤設計 | 「畫蛇添足 = 精益求精」(常見誤解) |
| 部分正確 | 選項看起來有道理但不完整 | 「因為他畫得太慢」(表面因果) |
| 相關但不同 | 使用同領域但意思不同的詞 | 「錦上添花」vs「畫蛇添足」(都有「添」) |
| 生活干擾 | 用學生熟悉但錯誤的生活經驗 | 「多做練習一定更好」(直覺但錯誤) |

## 5. 產出格式 (Output Format)

### 5.1 TypeScript 資料結構
```typescript
interface QuizQuestion {
    id: string;           // 格式: '{idiom-id}-p{n}' (practice) 或 '{idiom-id}-b{n}' (battle)
    question: string;     // 題目文字
    options: string[];    // 4 個選項 (含正確答案)
    answer: string;       // 正確答案 (字串比對)
    bloomLevel?: number;  // 1-6 對應布魯姆層次 (選填，用於分析)
}
```

### 5.2 產出範例
```typescript
// ✅ 正確範例：四個選項字數平衡 (約10字)
{
    id: 'hua-she-tian-zu-p1',
    question: '在故事中，第一個畫好蛇的人為什麼最後沒喝到酒？',
    options: [
        '因為他畫的蛇太醜被淘汰',     // 10字
        '因為他多做了幫蛇畫腳',       // 9字 ← 正確答案
        '因為他的蛇被人偷偷擦掉',   // 10字
        '因為他中途放棄不想比了'   // 10字
    ],
    answer: '因為他多做了幫蛇畫腳',
    bloomLevel: 1
}

// ❌ 錯誤範例：正確答案明顯最長
options: [
    '他生病了',           // 4字
    '他撿到一隻撞死的兔子，以為每天都會有',  // 15字 ← 正確答案
    '他的田被人搶走了',     // 8字
    '他找到更好的工作'      // 8字
]
// ↑ 學生只需「選最長的」就能答對，完全不需要理解內容
```

## 6. 品質檢核表 (Quality Checklist)

在產出題目後，**必須**逐題自我檢查以下項目：

- [ ] **認知層次覆蓋**：3 題中是否涵蓋了至少 L1 + L2 + L3？
- [ ] **正確答案唯一性**：是否有其他選項也可以被視為「正確」？
- [ ] **選項長度一致性**：最長和最短的選項字數差距是否超過 50%？
- [ ] **答案位置隨機性**：正確答案是否固定在同一個位置 (e.g. 全部都在 A)？
- [ ] **誘答項品質**：是否有選項一看就知道是錯的 (一眼假)？
- [ ] **題目獨立性**：是否有題目之間互相洩漏答案的情況？
- [ ] **年齡適切性**：用詞是否適合目標學齡？(國小中高年級)
- [ ] **無歧視內容**：是否包含性別、年齡、種族等歧視性語言？

## 7. 進階應用

### 7.1 難度調整 (Difficulty Scaling)
| 難度 | 題目特徵 |
|---|---|
| 簡單 (★) | L1 記憶題，答案在原文中可直接找到 |
| 中等 (★★) | L2 理解題，需要歸納或解釋文意 |
| 困難 (★★★) | L3 應用題，需要將概念遷移到全新情境 |
| 挑戰 (★★★★) | L4 分析題，需要拆解因果關係或比較異同 |

### 7.2 擴展至其他文本類型
本技能不僅適用於成語故事，也可擴展至：
- **記敘文** (故事、日記、新聞報導)
- **說明文** (科普文章、百科條目)
- **論說文** (社論、議題討論)
- **詩詞** (古詩、現代詩)

只需將 `文本內容` 替換為對應的閱讀材料，即可依照相同的框架產出題目。
